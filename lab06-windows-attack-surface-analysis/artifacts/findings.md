  - `#PermitRootLogin prohibit-password`

  - Relying on defaults makes hardening inconsistent across environments.
  - Explicit security baselines improve auditability and compliance.

- **Recommendation:**
  - Set explicit values in sshd_config and validate the effective config using:
    - `sshd -T`

---

### 2) SUID binaries present
- **Severity:** Medium  
- **Evidence (common SUID tools):**
  - `sudo`, `su`, `pkexec`, `passwd`, `mount`, `umount`, etc.

- **Why this matters:**
  - SUID increases privilege escalation risk if binaries are outdated/misconfigured.
  - Standard in Linux but should be reviewed regularly.

- **Recommendation:**
  - Remove unnecessary SUID binaries where possible.
  - Ensure patch levels are current.
  - Monitor changes using integrity checks (AIDE, auditd, osquery, etc.).
